# Roadmap

The roadmap is ordered by dependency and risk. A milestone is complete only when
its stated verification passes.

- [x] **M0 — Repository foundation.** Done when the private repository has documented invariants, local checks, and CI.
- [x] **M1 — Verified local citation vertical slice.** Done when a local UTF-8 source can be snapshotted, cited with an exact selector, and promoted only after hash and selector verification.
- [x] **M2 — Protocol adapter.** Done when promotion emits a version-pinned Sol Ledger event/artifact envelope without moving product-specific promotion rules into the protocol. Targets merged Sol Ledger commit `339dcf226440f5d135a16bbb7866ee7215ba6cfc`; JSON Schema and Rust/JCS compatibility are verified against a detached checkout.
- [x] **M3 — Durable local workspace.** Done when snapshots, candidates, evidence, and hash-linked append-only promotion records are queryable and crash-safe in a versioned local SQLite database.
- [x] **M4 — Web source capture.** Done when HTTP sources retain response bytes, canonical URL, retrieval metadata, and independently testable `availableAt` semantics.
- [x] **M5 — Review workspace.** Done when a reviewer can inspect source context, candidate status, and failed promotion reasons without hidden state.
- [x] **M6 — Full-stack dogfood and release baseline.** Done when Agent Black Box records a real capture→promotion run without content leakage, Sol Ledger verifies its trusted head, and the rendered Review Workspace is documented and responsive.
- [x] **M7 — Reproducible stack acceptance CLI.** Done when one shell-free command can run the three-product capture→promotion→trusted-head flow, reject retained private content, and emit a bounded machine-readable report.
- [x] **M8 — v0.2 failure fixtures and report review.** Done when unsafe trace/report variants fail closed and a reviewer can see a validated three-product result in the local Review Workspace on desktop and mobile.
- [x] **M9 — v0.3 history and regression matrix.** Done when up to 20 validated integration runs can be reviewed newest-first and every named unsafe fixture is exercised by a standalone machine-readable rejection matrix.
- [x] **M10 — v0.4 selectable history diff.** Done when a reviewer can select any of the eight visible runs and inspect per-product commit changes against its preceding run with keyboard-accessible controls.
- [x] **M11 — v0.5 versioned integrity bundle.** Done when stack reports have a published closed JSON Schema, new reports carry a verified JCS SHA-256, tampering fails closed, and legacy reports remain readable.
- [x] **M12 — v0.6 local trusted signatures.** Done when integrity-protected reports can be detached-signed with a 0600 Ed25519 key, verified only by explicitly trusted public keys, revoked by key ID, and shown without exposing key material.
- [x] **M13 — v0.7 quorum trust policy.** Done when Review Workspace requires an explicit N-of-M set of distinct trusted Ed25519 signers, rejects duplicate/revoked/out-of-window trust, and exposes only bounded verification metadata.
- [x] **M14 — Portable signed review bundle.** Done when a report, detached signatures, and public trust material can be exported and imported as one closed, path-free, size-bounded local package without private keys.
- [x] **M15 — Key rotation history.** Done when trust policy changes form a locally verifiable append-only chain and a reviewer can distinguish planned rotation from unexpected signer replacement.
- [x] **M16 — Standalone review verifier.** Done when one shell-free command verifies a bundle, optional rotation history, external anchors, report signatures, and active policy without starting the web workspace, then emits a bounded path-free receipt.
